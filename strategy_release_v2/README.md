# Block E: Momentum-Trend Strategy v2 (Ensemble + ML Overlay)

Это обновленная (v2) версия торговой стратегии Momentum-Trend для инструмента RTSF, которая включает **Ансамблирование**, **Фильтрацию рыночного режима (ADX)** и **AI-скоринг (ML Overlay) на базе XGBoost** с возможностью безопасного масштабирования через динамическое плечо.

## 🚀 Основные улучшения (относительно v1)

1. **Ансамблирование (Ensembles):** Вместо поиска одного лучшего набора параметров, мы запускаем 5 лучших наборов для LONG и 5 для SHORT. Общий капитал 5M делится по 1M на каждый набор. Это кардинально сглаживает кривую эквити и снижает дисперсию.
2. **ML-скоринг базовых сигналов (AI Overlay):** Поверх сигналов ансамбля работает XGBoost-модель. Она оценивает 50+ рыночных фичей в момент сигнала (сезонность, D1 тренды, объемы, микро-моментум) и предсказывает вероятность успеха. Если `ML probability < 0.5` — сделка игнорируется.
3. **Безопасное Плечо (Dynamic Leverage):** Поскольку ML-фильтр "срезал" просадки почти в 2-3 раза, на отфильтрованные качественные сигналы применяется плечо (до `x4.0`), возвращая абсолютную прибыль на сверх-высокий уровень при сохранении надежной кривой просадок.
4. **Режим Рынка (ADX D1):** Добавлена возможность пре-фильтрации слабых трендов с помощью ADX на дневном таймфрейме (с параметрами `adx_threshold` и `adx_action`).
5. **Partial Take-Profit (Scale-out):** Возможность закрыть часть позиции при достижении профита, равного N * ATR от точки входа.

## 🛠️ Инструкция по запуску Production Pipeline

### Шаг 1: Запуск Ансамбля и генерация базовых сделок
Этот скрипт загрузит Топ-5 параметров из папки `params/` (для лонга и шорта), прогонит по историческим M1 барам с базовым плечом `x2.0` и лимитами `cap_long=10`, `cap_short=4` (на 1 суб-систему) и сохранит список всех сделок.

```bash
python scripts/run_block_e_ensemble.py --ticker RTSF --data_path G:\data2 --train_start 2006-01-01 --train_end 2012-12-31 --test_start 2013-01-01 --test_end 2026-02-28
```
**Выход:** `outputs/experiments/block_e_ensemble_all_trades.csv`

### Шаг 2: Подготовка ML-датасета
Скрипт пройдет по всем сделкам ансамбля и в момент входа (`entry_time`) соберет макро- и микро-фичи (ADX, возвраты за 15м/60м, дневные объемы, день недели и т.д.).

```bash
python scripts/build_ml_dataset.py
```
**Выход:** `outputs/experiments/block_e_ml_dataset.csv`

### Шаг 3: Обучение модели (XGBoost)
Обучение XGBoost Classifier на тренировочном периоде (например, до конца 2012 года). Модель учится предсказывать `target` (PnL > 0) и сохраняет вероятности на OOS периоде (2013-2026) для проверки.

```bash
python scripts/train_ml_overlay.py
```
**Выход:**
- `outputs/experiments/block_e_xgboost_overlay.joblib` (Обученная модель)
- `outputs/experiments/block_e_ml_test_results.csv` (Предсказания на OOS)

### Шаг 4: Оценка и масштабирование плеча (ML + Leverage)
Скрипт объединяет OOS-сделки ансамбля с предсказаниями ML. Отсеивает сделки с `probability < 0.5`. Оставшиеся сделки прогоняются через Konkop Analyzer. Дополнительно (как было протестировано), можно применить повышающий коэффициент `contracts * 2.0` (эквивалентно плечу x4.0 от исходной базы), так как просадки теперь минимальны.

```bash
python scripts/evaluate_ml_impact.py
```
(Аналитика выводится прямо в консоль в формате Konkop Xpress)

## 📊 Результаты OOS (Out-Of-Sample 2013-2026, RTSF)

На длительном OOS (13 лет) ML-фильтр в комбинации с плечом `x4.0` показывает феноменальное превосходство над базовым ансамблем:

| Метрика | Базовый Ансамбль (x2.0, без ML) | ML-Ансамбль (x2.0) | ML-Ансамбль (x4.0) |
| :--- | :--- | :--- | :--- |
| **Сделки (Trades)** | 5501 | 2567 | **2567** (отфильтровано >50%) |
| **Net Profit (PnL)** | 1,921,788 | 1,637,205 | **3,274,410** (+70% к базе!) |
| **Max Drawdown %** | 21.53% | 11.16% | **18.99%** (всё равно ниже базы) |
| **Profit Factor** | 1.09 | 1.15 | **1.15** |
| **Sharpe Ratio (1c)** | 2.21 | 3.19 | **3.19** |
| **Ann.Ret / MaxDD** | 0.12 | 0.20 | **0.21** |

Модель блестяще "понимает" макро-контекст рынка (D1 тренды, объемы, сезонность) и отключает систему в "пиле", защищая капитал и позволяя безопасно масштабировать прибыльные сделки.
